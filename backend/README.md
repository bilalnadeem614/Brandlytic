Hey team, this is Wali. Here's what I've built for the backend and how you can connect your parts to it.

The backend is a Flask API that lives in the backend/ folder. The entry point is run.py, just run "python run.py" and the server starts on port 5000. Before running it you need to install the dependencies with "pip install -r requirements.txt" and create a .env file by copying .env.example and pasting your Groq API key into it. You can get a free Groq API key from console.groq.com.

There are two main endpoints.

The first is POST /api/v1/brand/generate. This is the main one. You send it a JSON body with three fields. The "input" field is required and can be either a plain text description of the business or a website URL, if it's a URL the backend automatically scrapes the site and extracts the text itself, so Flutter doesn't need to do anything special. The "platform" field is optional and can be "general", "instagram", "twitter", or "linkedin", this tells the AI to tailor the tagline and description for that specific platform, defaults to "general" if you don't send it. The "logo" field is optional boolean, set it to true if you want the AI to generate a logo concept description, defaults to false.

The response from /generate always comes back in the same structure. It will have a "success" field that is true or false. If true, the actual data is inside the "data" field. If false, the error message is in the "error" field. The data object contains brand_names (array of 3 suggestions), tagline, description, visual_identity (with primary_colors, font_style, overall_mood), logo_concept (string or null), marketing_content (with instagram_caption, elevator_pitch, slogan), target_audience, brand_values, and a "confidence" object. The confidence object has a score (0-100) and a short reason for every field, for example confidence.logo_concept.score might be 82 and confidence.logo_concept.reason might say "Strong visual alignment with the brand tone but color palette could be more distinct". The Flutter UI should display these scores next to each section so the user can see how confident the AI is.

The second endpoint is POST /api/v1/brand/refine. This is for the human-in-the-loop twist. When a user rejects a field, Flutter calls this endpoint instead of regenerating the whole brand kit. You send it five fields: "input" (same business description or URL as before), "platform" (same platform), "field" (which field the user is rejecting, must be one of brand_names, tagline, description, visual_identity, logo_concept, marketing_content, target_audience, brand_values), "rejected_value" (the actual value the user didn't like, as a string), and "rejection_reason" (what the user said was wrong with it). The backend sends all of this back to the AI which acknowledges the feedback and regenerates only that one field with an adjusted approach. The response contains "field", "acknowledgement" (a short sentence from the AI explaining what it understood and what it's changing), "value" (the new regenerated value), and "confidence" (updated score and reason for the new value). Flutter should display the acknowledgement message to the user so they feel heard, then update just that field in the UI.

There is also a health check at GET /api/v1/health that just returns ok, useful to ping before making real requests.

For the Firebase/Supabase integration, the backend doesn't touch storage at all, it just generates and refines brand kits and returns JSON. Whoever is handling storage should take the JSON response and save it on the client side. If you need a storage endpoint on the backend side later just let me know and I can add one.

There is a tests/ folder with tests you can run with "pytest tests/" after installing pytest via "pip install -r requirements-dev.txt". These tests don't hit the real Groq API so they run instantly.
