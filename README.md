# Brandlytic

AI-powered brand identity generator. Give it a business description or a website URL and it produces a complete brand kit — names, tagline, description, visual identity, marketing content, target audience, brand values, and an optional logo concept — with confidence scores for every field. You can reject any field and get a revised version with an acknowledgement of your feedback.

Built with a Flask backend (Python) and a Flutter frontend, using Google Gemini 2.0 Flash as the primary AI model with Groq (LLaMA 3.3 70B) as a fallback.

---

## Features

- Generate a full brand kit from a text description or a live website URL
- Platform-aware output tailored for General, Instagram, Twitter, or LinkedIn
- Confidence scores (0–100) with reasoning for every generated field
- Refine any field — the AI acknowledges your feedback and regenerates
- Logo image generation via Imagen 3 (Gemini)
- Automatic Gemini → Groq fallback if the primary model is unavailable

## Backend setup

### Prerequisites
- Python 3.10+
- A [Google Gemini API key](https://aistudio.google.com/app/apikey)
- A [Groq API key](https://console.groq.com/)

### Install and run

```bash
cd backend
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# Edit .env and fill in your GEMINI_API_KEY and GROQ_API_KEY

python run.py
# API is now running at http://localhost:5000
```

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/health` | Health check |
| POST | `/api/v1/brand/generate` | Generate a brand kit |
| POST | `/api/v1/brand/refine` | Regenerate a single field |
| POST | `/api/v1/brand/generate-logo` | Generate a logo image |

### Example: generate a brand kit

```bash
curl -X POST http://localhost:5000/api/v1/brand/generate \
  -H "Content-Type: application/json" \
  -d '{
    "input": "A subscription coffee box that sources beans from women-owned farms",
    "platform": "instagram",
    "logo": true
  }'
```

### Example: refine a field

```bash
curl -X POST http://localhost:5000/api/v1/brand/refine \
  -H "Content-Type: application/json" \
  -d '{
    "input": "A subscription coffee box that sources beans from women-owned farms",
    "platform": "instagram",
    "field": "tagline",
    "rejected_value": "Brewed with Purpose",
    "rejection_reason": "Too generic, I want something more specific to women farmers"
  }'
```

## Frontend setup

### Prerequisites
- Flutter SDK 3.x
- Firebase project with Authentication and Firestore enabled

```bash
cd frontend
flutter pub get
flutter run
```

> Make sure your Firebase config is in `lib/firebase_options.dart` (generated by `flutterfire configure`).

---

## Environment variables

| Variable | Description |
|----------|-------------|
| `GEMINI_API_KEY` | Google Gemini API key (primary AI model + logo generation) |
| `GROQ_API_KEY` | Groq API key (fallback AI model) |
| `FLASK_ENV` | `development` or `production` |
| `FLASK_DEBUG` | `1` to enable debug mode, `0` to disable |
